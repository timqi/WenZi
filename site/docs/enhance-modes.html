How It Works

AI enhancement is an optional post-processing step. When enabled, your transcribed text passes through an LLM before output:

Speech → ASR transcription → Enhancement mode (LLM) → Final text

Each enhancement mode is an independent Markdown file stored in ~/.config/VoiceText/enhance_modes/. The file contains a system prompt that tells the LLM how to process the text.

  • On startup, VoiceText ensures the built-in mode files exist. Missing ones are recreated; existing files are never overwritten.
  • All .md files in the directory are loaded and appear in the AI tab and mode switcher.
  • When a mode is active, the transcribed text is sent to the configured LLM with the mode's prompt as the system message.

Built-in Modes

These four modes are created automatically on first launch:

✎ Proofread (纠错润色)

ID: proofread Order: 10 Type: Single

Fix typos, grammar mistakes, and punctuation errors. Preserves the original meaning while polishing the text. The most universally useful mode — great as a default.

🌐 Translate to English (翻译为英文)

ID: translate_en Order: 20 Type: Single

Translate Chinese input into natural, fluent English. Handles ASR errors by inferring the intended meaning.

🔗 Proofread + Translate EN (润色+翻译EN)

ID: translate_en_plus Order: 25 Type: Chain (proofread → translate_en)

A chain mode that first proofreads the text, then translates the corrected text to English. Produces higher-quality translations by fixing errors before translating.

💻 Commandline Master (命令行大神)

ID: commandline_master Order: 30 Type: Single

Convert natural language descriptions into executable shell commands. Say "find all Python files larger than 1MB" and get the right find command.

File Format

Each mode is a .md file with YAML front matter followed by the prompt body:

---
label: Display Name
order: 50
---
System prompt content goes here.
You can use multiple lines.
FieldRequiredDescription
labelNoDisplay name in the menu. Defaults to the filename.
orderNoSort weight for menu ordering. Default 50. Lower = higher in menu.
stepsNoComma-separated mode IDs for chain execution (see Chain Modes).
bodyYesThe system prompt sent to the LLM. Everything after the second ---.

The filename (without .md) serves as the mode ID. Use only letters, numbers, hyphens, and underscores.

The reserved mode ID off disables enhancement and does not correspond to any file.

Chain Modes

A chain mode runs multiple enhancement steps sequentially, passing the output of each step as input to the next. This lets you combine existing modes into a pipeline without duplicating prompts.

How to Create

Add a steps field listing the mode IDs to execute in order:

---
label: Proofread + Translate EN
order: 25
steps: proofread, translate_en
---
First proofreads the text, then translates to English.
(This body is documentation only — each step uses its own prompt.)

How It Executes

  1. The input text is sent to the first step (proofread) using that mode's prompt.
  2. The output of step 1 becomes the input for step 2 (translate_en).
  3. The final output is the result of the last step.

UI Integration

  • Preview mode: Each step's output is displayed with a separator. The Final Result shows only the last step's output.
  • Direct mode: The streaming overlay shows step progress (e.g., "Step 1/2: 纠错润色") and updates in real-time.
The prompt body of a chain mode file is not sent to the LLM — each step uses its own mode's prompt. The body serves as documentation.

Create Your Own

Option A: From Settings (easy)

  1. Open Settings... → AI tab → Add Mode...
  2. Edit the template in the dialog.
  3. Click Save, enter a mode ID (e.g., summarize).
  4. The new mode appears in the menu immediately.

Option B: Create a File (flexible)

Create a new .md file in the modes directory:

~/.config/VoiceText/enhance_modes/summarize.md
---
label: Summarize
order: 55
---
You are a text summarization assistant.
Condense the user's input into a brief summary of 1-3 sentences.
Preserve the key information and original meaning.
Output only the summary without any explanation.

Restart the app to load the new mode.

Prompt Writing Tips

  1. Be specific. Tell the LLM exactly what to do AND what NOT to do. Vague prompts produce inconsistent results.
  2. End with an output rule. Always include a rule like "Output only the processed text without any explanation" to prevent the LLM from adding commentary.
  3. Mention ASR context. Add "The user's input comes from ASR and may contain recognition errors" — this significantly improves error correction.
  4. Use numbered rules. Structured prompts with numbered rules produce more consistent behavior than paragraph-form instructions.
  5. Set order with gaps. Use order values like 10, 20, 30... so you can insert modes between them later without renumbering.
  6. Test with edge cases. Try short input, long input, mixed languages, and intentionally noisy ASR text.
  7. Iterate. Start simple, test, then add rules for cases the LLM gets wrong. Over-constrained prompts can make the LLM too rigid.

Example: Well-Structured Prompt

---
label: Formal Email
order: 100
---
You are a professional email writing assistant.
The user's input comes from ASR and may contain recognition errors
— infer the intended meaning.

Rules:
1. Rewrite the input as a formal, polished email body
2. Add appropriate greeting and closing if the context suggests
   a standalone email
3. Maintain the original intent and key information
4. If the user mentions a recipient name, use it in the greeting
5. Output only the email text without any explanation

Manage Modes

Edit a Mode

Open the file directly with any text editor, or use Settings → AI → select the mode → edit. Changes take effect after restarting the app (or reloading config).

Built-in mode files can be freely edited. VoiceText will not overwrite a file that already exists.

Remove a Mode

Delete the corresponding .md file and restart. If you delete a built-in mode file (e.g., proofread.md), it will be recreated on the next startup with default content.

To permanently disable a built-in mode, replace its prompt with a passthrough instruction:

---
label: (Disabled) Proofread
order: 999
---
Output the user's input exactly as-is, without any changes.

Organize Modes

  • Non-.md files are ignored — you can safely keep notes (.txt) or backups (.bak) in the modes directory.
  • Config compatibility: If a mode file is removed but the config still references it, the app falls back to the first available mode.
  • Quick mode switch: Use ⌘1–⌘9 in the preview panel to switch modes on the fly.
